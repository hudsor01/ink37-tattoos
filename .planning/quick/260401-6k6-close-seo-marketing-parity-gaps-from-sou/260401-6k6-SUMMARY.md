# Quick Task 260401-6k6: Close SEO/marketing parity gaps - Summary

**Completed:** 2026-05-02

## Context

This task existed as an empty directory in `.planning/quick/` for an
unknown duration -- discovered during an end-of-session "incomplete
work" sweep on 2026-05-02. The slug suggests work to "close SEO /
marketing parity gaps from source [repo]" but no PLAN, SUMMARY, or
STATE files were ever written, and no source/reference repo was
identified anywhere in the planning artifacts.

Rather than delete the unstarted task, executed an SEO + marketing
audit of the current public surface and closed the actionable gaps.
The "source repo" framing is dropped (we audited against best
practices instead).

## Audit findings

A scan of the public surface (per-route metadata, root layout,
JSON-LD structured data, sitemap, robots, OpenGraph, Twitter cards,
performance, accessibility, local SEO, marketing parity, analytics)
identified gaps in three categories:

### BLOCKER

1. **No Twitter card metadata** -- root layout had `openGraph` but no
   `twitter` object. Twitter/X share previews fell back to basic link.
2. **No canonical URLs** -- none of the 7 public pages declared
   `alternates.canonical`. Risk of duplicate-content penalties when
   search engines see `/`, `/?utm=...`, `/?ref=...` etc. as separate.
3. **No Service schema** -- the four signature services (Custom,
   Traditional, Japanese, Realism) had no schema.org/Service markup,
   making them ineligible for service-listing rich results.

### IMPORTANT

4. **No standalone Person schema for Fernando Govea** -- only nested
   under `LocalBusiness.founder`, which doesn't qualify for
   Knowledge-Graph cards on artist-name searches.
5. **No `email` field on root LocalBusiness schema** -- contact email
   was in copy but not surfaced via schema, so it didn't appear in
   search-result contact extensions.
6. **No `sameAs` social profile link** on LocalBusiness -- the
   Instagram profile (`@fennyg83`) wasn't connected to the brand
   schema, weakening the entity graph.

### NICE-TO-HAVE (shipped in follow-up)

The first round of this task (PR #38) deferred the items below as
needing design assets, user decisions, or content. Most turned out
to have working solutions that didn't need any of those, so a
follow-up PR shipped them:

- **Per-page Open Graph images** -- shipped via `opengraph-image.tsx`
  in each route segment. Next.js's `next/og` `ImageResponse` API
  generates 1200x630 PNGs at request time from a React tree, so
  unique social previews don't require designer-produced PNG assets.
  Shared template in `src/lib/opengraph-template.tsx`; per-route
  files in `src/app/(public)/{,gallery,services,about,contact,faq,booking}/opengraph-image.tsx`.
- **Vercel Analytics + Speed Insights** -- shipped. Vercel-native,
  cookie-free, GDPR-friendly, free tier sufficient. No third-party
  signup; no tool-choice decision needed because we're already on
  Vercel. Wired in `src/components/providers.tsx`.
- **Conversion event tracking** -- shipped (minimum viable). The
  contact form's success path emits `track('contact_form_submitted')`
  via `@vercel/analytics`. Booking-page entries are auto-captured by
  Vercel's page-view tracking. Gallery-open events were considered
  but not shipped; they would be high-volume and low-signal.

### Still genuinely blocked (skipped)

- **Person.image** for Fernando Govea -- the schema's `image` field
  was DROPPED rather than pointing at a tattoo work (which would be
  misleading and could harm rich-result eligibility). When a real
  Fernando headshot is available, add it to the Person schema.
- **Review / AggregateRating schema** -- requires actual customer
  testimonials. Adding empty / fabricated review markup would be
  worse than not having any. Re-open this when the studio has
  collected and obtained permission to publish customer reviews.
- **ImageObject schema for portfolio pieces** -- marginal SEO value
  vs effort. Defer.

## What was changed

### `src/app/layout.tsx` (root)

- Added `twitter` metadata: `summary_large_image` card, brand title,
  description, default OG image as the Twitter image too.
- Added `alternates.canonical: '/'` so the root canonical resolves
  via `metadataBase` to the absolute URL.
- Added `email: 'contact@ink37tattoos.com'` to the LocalBusiness
  JSON-LD.
- Added `sameAs: ['https://instagram.com/fennyg83']` to the
  LocalBusiness JSON-LD so Instagram is connected to the brand
  entity.

### `src/app/(public)/{page,gallery,services,about,contact,faq,booking}/page.tsx`

- Added `alternates.canonical` to every public page's metadata
  export. Each canonical resolves via the root `metadataBase` to
  the absolute URL, so `/gallery` becomes
  `https://ink37tattoos.com/gallery`.

### `src/app/(public)/services/page.tsx`

- Added an `OfferCatalog` JSON-LD block listing the four signature
  services. Each `Service` carries:
    - `name` + `description`
    - `serviceType` (Custom Tattoo Design / Traditional Tattoo /
      Japanese Tattoo / Realism Tattoo)
    - `provider` reference to the TattooShop
    - `areaServed` (the same 5 cities the LocalBusiness covers)
    - `offers.priceSpecification` with min/max price ranges in USD,
      mirroring the price ranges shown in the page UI.

### `src/app/(public)/about/page.tsx`

- Added a standalone `Person` JSON-LD block for Fernando Govea with:
    - `name` / `givenName` / `familyName`
    - `jobTitle` and a comprehensive `description`
    - `image` (currently uses japanese.jpg as a placeholder; a
      headshot would be better when one is available)
    - `knowsAbout` listing the tattoo specialties
    - `sameAs` linking to the Instagram profile
    - `worksFor` linking back to the TattooShop

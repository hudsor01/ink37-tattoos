**Step 1: Create gallery page** at `src/app/(public)/gallery/page.tsx`
Server component that fetches all public designs and passes to client component.

```typescript
import type { Metadata } from 'next';
import { getPublicDesigns } from '@/lib/dal/designs';
import { GalleryClient } from '@/components/public/gallery-grid';

export const revalidate = 1800; // ISR: 30 minutes

export const metadata: Metadata = {
  title: 'Gallery | Ink 37 Tattoos',
  description: 'Browse our portfolio of custom tattoo art. Filter by style, placement, and size.',
  openGraph: { title: 'Gallery | Ink 37 Tattoos', description: 'Browse our portfolio of custom tattoo art.' },
};

export default async function GalleryPage() {
  const designs = await getPublicDesigns();
  return (
    <div className="max-w-7xl mx-auto px-4 md:px-6 py-12">
      <h1 className="text-2xl font-semibold mb-2">Gallery</h1>
      <p className="text-muted-foreground mb-8">Browse our portfolio of custom tattoo art</p>
      <GalleryClient initialDesigns={designs} />
    </div>
  );
}
```

Note: Fetches all designs server-side, filters client-side. This works for a single-studio portfolio (typically <500 images). Avoids the ISR+dynamic filter pitfall from research.

**Step 2: Create GalleryClient + GalleryGrid** at `src/components/public/gallery-grid.tsx`
'use client' component.

Filter categories (from UI spec):
- Style: Traditional, Neo-Traditional, Realism, Blackwork, Illustrative, Watercolor, Geometric, Japanese, Lettering, Minimalist
- Placement: Arm, Leg, Back, Chest, Shoulder, Ribs, Hand, Neck, Other
- Size: Small, Medium, Large, Extra Large

The component receives `initialDesigns` and manages filter state via `useSearchParams()` from `next/navigation`. Filters are stored in URL search params: `?style=Blackwork&placement=Arm&size=Small`.

Filtering logic:
- Read filters from URL search params
- Filter designs client-side: `style` matches `design.designType` or `design.style`, `placement` checks `design.tags` array, `size` matches `design.size`
- AND logic within group (only one value per group active at a time), OR across groups
- If no designs match filters, show empty state

Masonry layout using CSS `columns`:
```css
.masonry-grid {
  columns: 1;
  column-gap: 12px;
}
@media (min-width: 768px) { .masonry-grid { columns: 2; } }
@media (min-width: 1024px) { .masonry-grid { columns: 3; } }
```

Each grid item:
- `break-inside-avoid` to prevent column breaks
- `mb-3` for vertical gap
- Image wrapped in `next/image` with `width={400}` `height={0}` `style={{ height: 'auto' }}` for responsive aspect ratio
- On hover: scale(1.02) transform with 200ms ease-out transition (CSS, not Framer Motion -- per pitfall 6 about bundle size)
- On click: opens lightbox with the clicked image

Empty state (no designs at all):
```
Gallery Coming Soon
We're curating our portfolio. Check back soon or book a consultation to discuss your idea.
```

Empty state (no designs match filters):
```
No designs match your filters
Try adjusting your filter criteria to see more work.
```

**Step 4: Create CalEmbed component** at `src/components/public/cal-embed.tsx`
'use client' component.

```typescript
'use client';
import Cal, { getCalApi } from '@calcom/embed-react';
import { useEffect } from 'react';

interface CalEmbedProps {
  calLink: string;
}

export function CalEmbed({ calLink }: CalEmbedProps) {
  useEffect(() => {
    (async () => {
      const cal = await getCalApi();
      cal('ui', {
        theme: 'light',
        styles: { branding: { brandColor: '#e8432b' } },
        hideEventTypeDetails: false,
      });
    })();
  }, []);

  return (
    <Cal
      calLink={calLink}
      style={{ width: '100%', height: '100%', overflow: 'auto' }}
      config={{ layout: 'month_view' }}
    />
  );
}
```

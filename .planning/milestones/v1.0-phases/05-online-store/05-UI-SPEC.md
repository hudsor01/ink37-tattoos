## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, price text spacing |
| sm | 8px | Compact element spacing, cart item gaps |
| md | 16px | Default element spacing, card padding, product grid gap |
| lg | 24px | Section padding, form field groups |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Major section breaks (store page top/bottom) |
| 3xl | 64px | Page-level vertical spacing |

Exceptions:
- Product grid gap: 24px (lg) on desktop, 16px (md) on mobile
- Cart drawer width: 384px (sm:max-w-sm from existing Sheet component)
- Mobile touch targets: minimum 44px height for add-to-cart buttons, quantity controls, filter pills
- Gift card denomination buttons: 48px minimum height for prominent selection targets

Source: Inherited from Phase 2 spacing scale; exceptions specific to store interactions.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (1rem) | 400 (regular) | 1.5 | Product descriptions, cart item names, gift card message |
| Label | 14px (0.875rem) | 400 (regular) | 1.4 | Product type filter pills, cart meta text, order table headers, price labels, shipping info |
| Heading | 24px (1.5rem) | 600 (semibold) | 1.2 | Store page title, product detail name, checkout headings, admin product/order page titles |
| Display | 40px (2.5rem) | 600 (semibold) | 1.1 | Not used in Phase 5 -- store pages use Heading (24px) as top-level |

Notes:
- Product card titles use Body (16px) at weight 600 (semibold) for scannable names in the grid
- Product prices render at Label size (14px) at weight 600 (semibold) for visual weight
- Gift card denomination labels use Body (16px) at weight 600 (semibold) inside selection buttons
- Cart total and subtotal use Heading (24px) at weight 600 (semibold)
- Display (40px) is not used because the store is accessed through the public site nav and does not need a hero-scale headline

Source: Inherited from Phase 2 typography scale.

## Color

The project uses shadcn CSS variables with oklch color space. The store extends the established palette without adding new colors.

### Light Mode

| Role | CSS Variable | oklch Value | Hex Approx | Usage |
|------|-------------|-------------|------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) | #ffffff | Store page background, product detail background, cart drawer background |
| Secondary (30%) | `--card`, `--secondary` | oklch(1 0 0), oklch(0.97 0 0) | #ffffff, #f5f5f5 | Product cards, gift card denomination buttons (inactive), filter pill (inactive), cart summary section |
| Primary/CTA (10%) | `--primary` | oklch(0.205 0 0) | #1a1a1a | "Add to Cart" button, "Proceed to Checkout" button, active filter pill background |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) | #dc2626 | Remove from cart button, cancel order (admin), refund action (admin) |

### Brand Accent (Store Public Pages)

| Token | oklch Value | Hex Approx | Reserved For |
|-------|-------------|------------|--------------|
| `--brand-accent` | oklch(0.637 0.237 25.331) | #e8432b | Store nav "Book a Consultation" CTA (inherited from public nav), active type filter pill, gift card "Send Gift Card" CTA button |

Accent reserved for (explicit list):
1. "Book a Consultation" CTA in store header nav (inherited from PublicNav)
2. Active state of product type filter pills (Merch / Prints / Gift Cards)
3. "Send Gift Card" CTA button on gift card purchase page
4. Social media icon hover in footer (inherited from PublicFooter)

Store product interaction buttons ("Add to Cart", "Proceed to Checkout") use `--primary` (dark neutral), NOT brand-accent. This keeps the store's conversion flow visually distinct from the booking CTA and matches the admin dashboard's neutral button pattern.

## Layout Contracts

### Store Layout (Public-Facing)

```
+------------------------------------------+
|  StoreNav (sticky, h-16, z-50)           |
|  Logo left | Nav center | Cart+CTA right |
+------------------------------------------+
|                                          |
|  Page Content (max-w-7xl, mx-auto,       |
|  px-16px mobile / px-24px desktop)       |
|                                          |
+------------------------------------------+
|  PublicFooter                            |
|  3-column grid on desktop, stack mobile  |
+------------------------------------------+
```

- Store shares the PublicNav pattern with an added "Shop" link and CartIcon
- CartIcon sits to the left of the "Book a Consultation" CTA button
- Cart count badge renders only after client-side hydration (avoids SSR mismatch)
- Content max-width: 1280px (max-w-7xl) -- matches public site
- Page sections separated by 48px (2xl) vertical spacing
- Primary visual anchor: product grid with type filter above it

### Store Page (Product Catalog)

```
+------------------------------------------+
|  Heading: "Shop"                          |

|                                          |
|  [All] [Merch] [Prints] [Gift Cards]    |
|                                          |
|  +--------+ +--------+ +--------+       |
|  | Image  | | Image  | | Image  |       |

|  +--------+ +--------+ +--------+       |
|  +--------+ +--------+ +--------+       |

|  +--------+ +--------+ +--------+       |

```

- Product grid: 1 column mobile, 2 columns tablet (md), 3 columns desktop (lg)
- Grid gap: 24px (lg) on all breakpoints

- Filter pills: rounded-full, 14px label text, 32px height, 12px horizontal padding
- Active filter pill: `--brand-accent` background, white text
- Inactive filter pill: `--secondary` background, `--secondary-foreground` text

## Interaction Patterns

### Cart Interactions

| Interaction | Behavior |
|-------------|----------|
| Add to cart (product detail) | Item added to Zustand store, CartDrawer opens automatically, cart icon badge updates |
| Add to cart (duplicate item) | Quantity incremented, CartDrawer opens, brief highlight on updated item |
| Remove from cart | Item removed immediately (no confirmation -- low-risk action, undo not needed) |
| Update quantity | Instant update via +/- buttons; quantity of 0 removes item |
| Open cart | Click CartIcon in nav opens Sheet from right |
| Proceed to checkout | Redirects to Stripe Checkout hosted page; cart persists in localStorage until webhook confirms order |
| Cart cleared | Only after successful Stripe webhook confirms payment (not on redirect to Stripe) |

### Gift Card Flow

| Interaction | Behavior |
|-------------|----------|
| Select denomination | Single selection -- clicking one deselects others, highlight with primary background |
| Form validation | Inline field-level errors; recipient email validates format; all fields required except personal message |
| Submit | Creates Stripe Checkout session for gift card amount; redirects to Stripe |
| Success | Gift card code generated, email sent to recipient, success page shown |

### Product Type Filter

| Interaction | Behavior |
|-------------|----------|
| Click filter pill | Filters product grid client-side (no URL params -- small catalog). Uses React state. |
| Active filter | Brand-accent background, white text |
| "All" selected | Shows all products, "All" pill is active |
| Transition | Products fade/filter instantly (no animation delay for <20 items) |

### Loading States

| Surface | Pattern |
|---------|---------|
| Product grid | Skeleton cards matching grid layout (3 columns, aspect-1/1 image placeholder + text lines) |
| Product detail | Skeleton with image placeholder left, text lines right |
| Cart drawer | Not needed -- cart is client-side state, instant |
| Admin product table | Skeleton rows (5 rows) -- same pattern as existing admin tables |
| Admin order table | Skeleton rows (5 rows) -- same pattern as existing admin tables |

### Error States

| Surface | Pattern |
|---------|---------|
| Product data fetch failure | Inline error message with "Try Again" button (matches Phase 2 pattern) |
| Add to cart failure | Not possible -- client-side only |
| Checkout redirect failure | Sonner toast: "Couldn't start checkout. Please try again." |
| Gift card validation failure | Inline field error below code input |
| Admin mutation failure | Sonner toast (error variant) with retry suggestion |
| Download link expired | Full-page message with expired heading + body (HTTP 410 response) |

### Empty States

| Surface | Pattern |
|---------|---------|
| Store (no products) | Centered message: "Shop Coming Soon" heading + body text |
| Filtered results (no matches) | Centered message: "No products match this filter" + suggestion |
| Cart (empty) | Centered in drawer: "Your cart is empty" heading + body + shop link |
| Admin products (none) | Centered: "No products yet" + "Add Product" CTA |
| Admin orders (none) | Centered: "No orders yet" + explanatory body |

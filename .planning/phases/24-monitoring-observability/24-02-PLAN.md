<!-- Current console.* patterns that will be replaced -->

Pattern 1 - Tagged errors (most common):
```typescript
console.error('[Cal Webhook] Payload validation failed:', { errors });
// becomes: log.error({ errors }, 'Payload validation failed')
```

Pattern 2 - Error with context object:
```typescript
console.error('Gift card email delivery failed:', emailError);
// becomes: log.error({ err: emailError }, 'Gift card email delivery failed')
```

Pattern 3 - Warnings for missing config:
```typescript
console.warn('[Email] RESEND_API_KEY not configured');
// becomes: log.warn('RESEND_API_KEY not configured')
```

Pattern 4 - Info logging:
```typescript
console.log(`Cal.com ${event.triggerEvent}: ${payload.uid}`);
// becomes: log.info({ triggerEvent: event.triggerEvent, uid: payload.uid }, 'Cal.com webhook processed')
```

    2. Create `src/lib/logger.ts` with 'server-only' directive:
       ```
       import 'server-only';
       import pino from 'pino';

       export const logger = pino({
         level: process.env.LOG_LEVEL || (process.env.NODE_ENV === 'production' ? 'info' : 'debug'),
         // In production on Vercel, plain JSON is ideal -- Vercel parses structured logs natively
         // In development, use pino's default JSON (can pipe to pino-pretty locally if desired)
         formatters: {
           level: (label) => ({ level: label }),
         },
         timestamp: pino.stdTimeFunctions.isoTime,
       });

       export function createLogger(module: string) {
         return logger.child({ module });
       }

       export type Logger = pino.Logger;
       ```

       **Conversion rules:**
       - `console.error('message:', error)` -> `log.error({ err: error }, 'message')`
       - `console.error('message:', { obj })` -> `log.error({ ...obj }, 'message')`
       - `console.warn('message')` -> `log.warn('message')`
       - `console.log('message')` -> `log.info('message')`
       - String interpolation like `console.log(\`prefix ${var}\`)` -> `log.info({ var }, 'prefix description')`
       - IMPORTANT: In Pino, the FIRST argument is the context object (or error), the SECOND is the message string. Never put the error as the second argument.
       - IMPORTANT: Use `err` as the key name for Error objects (Pino serializes Error objects specially when the key is `err`)

       **Specific high-value conversions:**

       In `src/app/api/webhooks/stripe/route.ts`:
       - `console.error('Webhook signature verification failed:', err)` -> `log.error({ err }, 'Webhook signature verification failed')`
       - `console.error(\`Webhook handler error for ${event.type}:\`, err)` -> `log.error({ err, eventType: event.type }, 'Webhook handler error')`
       - `console.error('Checkout session missing tattooSessionId in metadata')` -> `log.error({ sessionId: session.id }, 'Checkout session missing tattooSessionId in metadata')`

       In `src/app/api/webhooks/cal/route.ts`:

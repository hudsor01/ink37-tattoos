    2. **Consent DAL** (`src/lib/dal/consent.ts`): Create server-only DAL with `requireStaffRole` guard:
       - `getConsentForms()` -- returns all consent form versions ordered by version desc
       - `getActiveConsentForm()` -- returns the latest active form (isActive=true)
       - `getConsentFormsByCustomer(customerId: string)` -- returns tattooSessions for that customer with consent fields (consentSigned, consentSignedAt, consentSignedBy, consentFormVersion, consentExpiresAt)
       - `getSignedConsentsWithExpiration({ page, pageSize, search, filter })` -- paginated query joining tattooSession + customer. Filter: 'all' | 'active' | 'expired' | 'pending'. Active = consentSigned && consentExpiresAt > now. Expired = consentSigned && consentExpiresAt <= now. Pending = !consentSigned. Search on customer name.
       - `createConsentFormVersion({ title, content })` -- inside a transaction: get max version, deactivate all existing active versions, insert new version = max+1 with isActive=true. Return the new form.

    3. **Consent actions** (`src/lib/actions/consent-actions.ts`): Create with `requireRole('admin')`:
       - `createConsentFormVersionAction(formData: FormData)` -- validates title + content, calls createConsentFormVersion DAL, revalidates '/dashboard/consent', audit logs
       - `deactivateConsentFormAction(id: string)` -- sets isActive=false, revalidates, audit logs

    4. **Validation schemas** (`src/lib/security/validation.ts`): Add `ConsentFormSchema` for creating consent form versions (title: string min 1 max 200, content: string min 10).

    5. **Env schema** (per research pitfall 6): In `src/lib/env.ts`, add `CRON_SECRET: z.string().optional()` to the envSchema object.

    1. **Invoice template** (`src/lib/invoice-template.ts`, per D-09 BIZ-05): Mirror `receipt-template.ts` structure exactly. Import `escapeHtml` from `./receipt-template`, import `format` from `date-fns`.

       Define interface:
       ```typescript
       export interface InvoiceData {
         studioName: string;
         customerName: string;
         customerEmail?: string;
         invoiceNumber: string;
         invoiceDate: Date;
         dueDate: Date;
         lineItems: Array<{ description: string; amount: number }>;
         subtotal: number;
         depositPaid: number;
         totalDue: number;
         terms?: string;
       }
       ```

       `renderInvoiceHtml(data: InvoiceData): string` -- HTML document with inline CSS (same styling approach as receipt). Sections:
       - Header: Studio name, "INVOICE" title, invoice number, invoice date, due date
       - Bill To: Customer name, email
       - Line items table: Description | Amount columns. Each line item row. Subtotal row. Deposit paid row (if > 0, shown as negative). Total due row (bold).
       - Terms section (if provided): rendered below the table
       - Footer: "Thank you for your business" + studio contact info
       - Invoice number format: `INV-{first 8 chars of paymentId uppercase}`

    2. **Invoice PDF route** (`src/app/api/invoices/[paymentId]/route.ts`, per D-09 BIZ-05): Mirror `receipts/[paymentId]/route.ts` almost exactly:
       - Auth check via `getCurrentSession()`
       - Fetch payment via `getPaymentWithDetails(paymentId)`
       - Require payment.status === 'COMPLETED'
       - Build InvoiceData from payment details:
         - invoiceNumber: `INV-${paymentId.slice(0, 8).toUpperCase()}`
         - invoiceDate: payment.completedAt ?? payment.createdAt
         - dueDate: same as invoiceDate (already paid)
         - lineItems: `[{ description: sessionDescription, amount: payment.amount }]`
         - subtotal: payment.tattooSession?.totalCost ?? payment.amount
         - depositPaid: payment.tattooSession?.depositAmount ?? 0
         - totalDue: payment.amount
         - terms: fetch from settings `getSettingByKey('invoice_terms')` -- use value if set, otherwise default "Payment received. Thank you."
       - Call `renderInvoiceHtml(invoiceData)`
       - Health check HEAD to STIRLING_PDF_URL with 5s timeout (same 503 pattern)
       - POST HTML to Stirling PDF with FormData + 15s timeout
       - Return PDF blob with `Content-Type: application/pdf` and `Content-Disposition: attachment; filename="invoice-${paymentId.slice(0, 8)}.pdf"`

    3. **Invoice email action** (`src/lib/actions/invoice-actions.ts`, per D-09 BIZ-05):
       - `emailInvoiceAction(paymentId: string)` with `requireRole('admin')`
       - **IMPORTANT: Generate PDF inline -- do NOT fetch from the internal invoice API route.** This avoids auth forwarding complexity and cookie propagation issues in server actions.
       - Inside safeAction wrapper:
         - Fetch payment via `getPaymentWithDetails(paymentId)`
         - Verify payment exists and is COMPLETED
         - Get customer email from payment.customer.email -- if no email, return error
         - Build InvoiceData (same logic as the route handler in step 2)
         - Call `renderInvoiceHtml(invoiceData)` to get HTML string
         - POST the HTML to Stirling PDF directly (same STIRLING_PDF_URL, FormData, timeout pattern as the route):
           ```typescript
           const STIRLING_PDF_URL = 'https://pdf.thehudsonfam.com/api/v1/convert/html/pdf';
           const formData = new FormData();
           formData.append('fileInput', new Blob([html], { type: 'text/html' }), 'invoice.html');
           formData.append('zoom', '1');
           const pdfResponse = await fetch(STIRLING_PDF_URL, {
             method: 'POST',
             body: formData,
             signal: AbortSignal.timeout(15000),
           });
           ```
         - Convert PDF response to Buffer: `Buffer.from(await pdfResponse.arrayBuffer())`
         - Call `sendInvoiceEmail({ to: customer.email, customerName, invoiceNumber, totalDue, pdfBuffer })`
         - Audit log the email send
         - Create admin notification
         - revalidatePath for the payment page
       - Return `{ success: true, data: { sent: true } }`

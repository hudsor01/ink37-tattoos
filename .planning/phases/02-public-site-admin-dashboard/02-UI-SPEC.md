# Phase 2 — UI Design Contract

> Visual and interaction contract for the public site and admin dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (base-nova preset) |
| Preset | base-nova, neutral base color |
| Component library | base-ui (via shadcn v2) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tight label spacing |
| sm | 8px | Compact element spacing, form field gaps |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, sidebar item spacing |
| xl | 32px | Layout gaps between major content blocks |
| 2xl | 48px | Major section breaks (public site sections) |
| 3xl | 64px | Page-level vertical spacing, hero padding |

Exceptions:
- Mobile touch targets: minimum 44px height for interactive elements (buttons, nav links, form inputs)
- Gallery grid gap: 12px (3 * 4px) for tighter masonry layout
- Admin sidebar width: 256px collapsed icon-only: 64px

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (1rem) | 400 (regular) | 1.5 | Paragraphs, descriptions, form labels |
| Label | 14px (0.875rem) | 400 (regular) | 1.4 | Table headers, badges, meta text, nav items, small UI text |
| Heading | 24px (1.5rem) | 600 (semibold) | 1.2 | Page titles, section headers, card titles |
| Display | 40px (2.5rem) | 600 (semibold) | 1.1 | Hero headline, public site section titles |

Notes:
- Admin dashboard uses Body, Label, and Heading only (no Display)
- Public site uses all four roles
- Display size reduces to 32px on screens below 768px
- All text renders in Inter; no decorative/script fonts in v1

## Color

The project uses shadcn CSS variables with oklch color space. The default neutral palette from `base-nova` is the foundation. A brand accent color is added for CTAs and key interactive elements.

### Light Mode (primary -- no dark mode in v1)

| Role | CSS Variable | oklch Value | Hex Approx | Usage |
|------|-------------|-------------|------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) | #ffffff | Page backgrounds, main content area |
| Secondary (30%) | `--card`, `--secondary` | oklch(1 0 0), oklch(0.97 0 0) | #ffffff, #f5f5f5 | Cards, sidebar, table rows, nav background |
| Primary/CTA (10%) | `--primary` | oklch(0.205 0 0) | #1a1a1a | Primary buttons, active nav, key interactive elements |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) | #dc2626 | Delete buttons, error states, cancel actions |

### Brand Accent (Public Site Only)

| Token | oklch Value | Hex Approx | Reserved For |
|-------|-------------|------------|--------------|
| `--brand-accent` | oklch(0.637 0.237 25.331) | #e8432b | Public site booking CTA, hero CTA button, gallery hover overlay, active filter pill |

Accent reserved for (explicit list):
1. "Book a Consultation" CTA button on public pages (hero, nav, services)
2. Gallery image hover overlay border/indicator
3. Active state of gallery filter pills
4. "Send Message" contact form submit button
5. Public site footer social media icon hover

Admin dashboard does NOT use brand accent -- it uses the neutral `--primary` (dark) for all interactive elements, keeping the admin clean and professional.

### Semantic Colors

| State | Color | Usage |
|-------|-------|-------|
| Success | oklch(0.627 0.194 145.071) / #16a34a | Toast confirmations, status badges (completed), positive KPI indicators |
| Warning | oklch(0.769 0.189 70.08) / #eab308 | Pending status badges, attention indicators |
| Info | oklch(0.623 0.214 259.815) / #2563eb | Informational toasts, link text on admin |
| Error | oklch(0.577 0.245 27.325) / #dc2626 | Form validation errors, failed status, destructive actions |

## Layout Contracts

### Public Site Layout

```
+------------------------------------------+
|  PublicNav (sticky, h-16, z-50)          |
|  Logo left | Nav center | CTA right     |
+------------------------------------------+
|                                          |
|  Page Content (max-w-7xl, mx-auto,       |
|  px-16px mobile / px-24px desktop)       |
|                                          |
+------------------------------------------+
|  PublicFooter                            |
|  3-column grid on desktop, stack mobile  |
+------------------------------------------+
```

- Nav becomes hamburger menu below 768px (sheet/drawer pattern)
- Content max-width: 1280px (max-w-7xl)
- Page sections separated by 48px (2xl) vertical spacing
- Hero section is full-bleed (no max-width constraint)

### Admin Dashboard Layout

```
+------+----------------------------------+
| Side |  Breadcrumb bar (h-12)           |
| bar  +----------------------------------+
| 256px|                                  |
|      |  Page Content                    |
|      |  (p-24px, gap-24px)             |
|      |                                  |
+------+----------------------------------+
```

- Sidebar collapses to icon-only (64px) below 1024px
- Sidebar hidden on mobile, accessible via sheet/drawer toggle
- Content area has no max-width (fills available space)
- Admin pages use consistent 24px padding and 24px gap between sections

## Interaction Patterns

### Animations (Framer Motion)

| Pattern | Duration | Easing | Usage |
|---------|----------|--------|-------|
| Fade in | 300ms | ease-out | Page content entrance, card appearance |
| Slide up + fade | 400ms | [0.25, 0.1, 0.25, 1] | Hero text entrance, section reveals on scroll |
| Scale on hover | 200ms | ease-out | Gallery image hover (scale 1.02) |
| Stagger children | 50ms delay each | -- | Gallery grid items, service cards |
| Skeleton pulse | 1.5s loop | ease-in-out | Loading states (CSS animation, not Framer) |

### Loading States

| Surface | Pattern |
|---------|---------|
| Gallery grid | Skeleton rectangles matching masonry layout |
| Admin tables | Skeleton rows (5 rows of varying width blocks) |
| KPI cards | Skeleton with pulse animation |
| Full page | Centered spinner is NOT used; always use skeleton/placeholder |

### Error States

| Surface | Pattern |
|---------|---------|
| Data fetch failure | Inline error message with "Try Again" button (not toast) |
| Form validation | Inline field-level errors below each input, red border on invalid fields |
| Mutation failure | Sonner toast (error variant) with retry suggestion |
| Network offline | Not handled in v1 |

### Empty States

| Surface | Pattern |
|---------|---------|
| Admin tables | Centered illustration-free message with heading + body + primary CTA button |
| Gallery (no images) | Full-section centered message with heading + body |
| Analytics (no data) | Card with muted message, no chart rendered |

## Gallery Filtering

Categories (based on tattoo industry standards):

| Filter Group | Options |
|-------------|---------|
| Style | Traditional, Neo-Traditional, Realism, Blackwork, Illustrative, Watercolor, Geometric, Japanese, Lettering, Minimalist |
| Placement | Arm, Leg, Back, Chest, Shoulder, Ribs, Hand, Neck, Other |
| Size | Small (< 3"), Medium (3-6"), Large (6-12"), Extra Large (12"+) |

- Filters render as horizontal scrollable pill buttons
- Multiple filters can be active simultaneously (AND logic within group, OR across groups)
- Active filter pill uses `--brand-accent` background with white text
- Inactive filter pill uses `--secondary` background
- "All" option clears all filters in that group
- Filter state managed via URL search params for shareability

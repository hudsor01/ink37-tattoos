    Per D-08: Enhance analytics page with date range picker, export, period comparison, and new KPIs.

    **New DAL functions in src/lib/dal/analytics.ts:**

    1. `getAnalyticsKPIs(from: Date, to: Date)` -- cached, requireStaffRole. Computes:
       - **CLV (Customer Lifetime Value):** Total revenue / total unique customers with completed sessions. Use SQL: `sum(totalCost) / nullif(count(distinct customerId), 0)` from tattooSession joined with appointment.
       - **No-show rate:** Count of appointments with status='NO_SHOW' / total appointments in period. Use SQL count with CASE.
       - **Avg session duration:** Average of estimatedHours from tattooSession where status='COMPLETED' in period. If estimatedHours is null for most sessions, fall back to counting sessions per appointment as a proxy.
       Return: `{ clv: number, noShowRate: number, avgSessionDuration: number }`

    2. `getAnalyticsDataByDateRange(from: Date, to: Date)` -- cached, requireStaffRole. Calls existing functions (getRevenueByDateRange, getClientAcquisitionData, getAppointmentTypeBreakdown, getBookingTrends) but with date range parameters where possible. For functions that take `months`, compute months from date range. Returns combined analytics data object.

    3. `getComparisonPeriodData(from: Date, to: Date)` -- cached, requireStaffRole. Calculates the previous period of equal length, fetches KPIs for both periods, returns both with trend percentages. Similar to getDashboardStatsWithTrend but for analytics KPIs.

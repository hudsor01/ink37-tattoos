    2. Add `searchVector` generated column to these 8 tables (per research tsvector column recommendations). Use `sql.raw()` for column references inside generated columns to avoid self-referencing issues:

    | Table | Columns | Weights |
    |-------|---------|---------|
    | customer | firstName + lastName (A), email (B), phone (C) | |
    | appointment | firstName + lastName (A), email (B), description + notes (C), type (D) | |
    | tattooSession | designDescription (A), placement + style (B), notes (C) | |
    | contact | name (A), email (B), message (C) | |
    | product | name (A), description (B) | |
    | tattooDesign | name (A), description + style (B) | |
    | order | email (A), shippingName (B) | |
    | auditLog | action + resource (A) | |

    For each table, add a generated column like this pattern:
    ```typescript
    searchVector: tsvector('search_vector').generatedAlwaysAs(
      sql`setweight(to_tsvector('english', coalesce("first_name", '') || ' ' || coalesce("last_name", '')), 'A') || setweight(to_tsvector('english', coalesce("email", '')), 'B')`
    ),
    ```
    NOTE: Use the actual SQL column names (snake_case) inside the sql template, NOT the Drizzle camelCase names. This avoids the self-referencing pitfall (Pitfall 5 from research).

    3. Add GIN index for each table's searchVector. Add to the table's index array:
    ```typescript
    index('customer_search_idx').using('gin', table.searchVector),
    ```

    **Part B: Analytics refactoring (per DAL-03)**

    Refactor `getRevenueData`, `getClientAcquisitionData`, and `getBookingTrends` in analytics.ts to use SQL `GROUP BY` with `date_trunc()` instead of fetching all rows and aggregating in JavaScript.

    `getRevenueData` should use:
    ```sql
    SELECT to_char(date_trunc('month', "appointment_date"), 'YYYY-MM') as month,
           coalesce(sum("total_cost"), 0) as revenue,
           cast(count(*) as integer) as count
    FROM "tattoo_session"
    WHERE status = 'COMPLETED' AND "appointment_date" >= $startDate
    GROUP BY date_trunc('month', "appointment_date")
    ORDER BY date_trunc('month', "appointment_date")
    ```

    `getClientAcquisitionData` should use:
    ```sql
    SELECT to_char(date_trunc('month', "created_at"), 'YYYY-MM') as month,
           cast(count(*) as integer) as count
    FROM "customer"
    WHERE "created_at" >= $startDate
    GROUP BY date_trunc('month', "created_at")
    ORDER BY date_trunc('month', "created_at")
    ```

    `getBookingTrends` should use:
    ```sql
    SELECT to_char(date_trunc('week', "scheduled_date"), 'YYYY-MM-DD') as week,
           cast(count(*) filter (where status NOT IN ('CANCELLED', 'NO_SHOW')) as integer) as bookings,
           cast(count(*) filter (where status IN ('CANCELLED', 'NO_SHOW')) as integer) as cancellations
    FROM "appointment"
    WHERE "scheduled_date" BETWEEN $startDate AND $endDate
    GROUP BY date_trunc('week', "scheduled_date")
    ORDER BY date_trunc('week', "scheduled_date")
    ```

    Use Drizzle's `sql` template literal for these queries (the relational API does not support aggregations -- per CLAUDE.md pitfall).

    After schema changes, run `npx drizzle-kit generate` to generate the migration, then `npx drizzle-kit push` to apply (or note in summary that migration needs applying to the database).

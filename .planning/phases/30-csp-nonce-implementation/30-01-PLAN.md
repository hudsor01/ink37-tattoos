<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Server | CSP header tells browser which scripts to trust; nonce is the trust token |
| proxy.ts -> layout.tsx | x-nonce request header passes nonce from middleware to server components |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-01 | Tampering | Inline script injection (XSS) | mitigate | Nonce-based script-src in CSP; only scripts with valid nonce execute; `'strict-dynamic'` covers dynamically loaded chunks |
| T-30-02 | Tampering | Plugin-based attacks (Flash/Java) | mitigate | `object-src 'none'` blocks all plugin content |
| T-30-03 | Spoofing | Clickjacking via iframe embedding | mitigate | `frame-ancestors 'none'` in CSP (supplements existing X-Frame-Options: DENY in securityHeaders) |
| T-30-04 | Information Disclosure | Data exfiltration via unauthorized fetch | mitigate | Restrictive `connect-src` limits fetch/XHR to self + Sentry + Cal.com only |
| T-30-05 | Information Disclosure | CSS exfiltration via injected styles | accept | `'unsafe-inline'` required in style-src due to Next.js route announcer and chart.tsx inline styles; nonce-based styles not feasible in current Next.js architecture (GitHub issues #18557, #83764) |
| T-30-06 | Tampering | Weak nonce generation (predictable) | mitigate | Uses `crypto.randomUUID()` (Node.js CSPRNG) not Math.random(); base64-encoded for CSP compatibility |
| T-30-07 | Denial of Service | CSP report-uri flood | accept | No report-uri configured; CSP violations are client-side only; no server-side reporting endpoint to overwhelm |
</threat_model>

**src/app/api/store/download/route.ts** -- Route handler for secure digital downloads (per D-15, D-16, Research Pattern 5):
```typescript
import { type NextRequest, NextResponse } from 'next/server';
import { db } from '@/lib/db';
```
- GET handler:
  1. Extract `token` from searchParams. If missing: return 400 "Missing token"
  2. Query `db.downloadToken.findUnique({ where: { token }, include: { orderItem: { include: { product: true } } } })`
  3. If not found: return 404 "Invalid token"
  4. If `expiresAt < new Date()`: return 410 JSON `{ error: 'Download link expired' }` (per UI-SPEC: "Download Link Expired")
  5. If `downloadCount >= maxDownloads`: return 410 "Download limit reached"
  6. Get the product's `digitalFilePathname` -- if null, return 404
  7. Use `@vercel/blob` `get()` with `access: 'private'` to fetch the file using BLOB_PRIVATE_READ_WRITE_TOKEN
  8. Increment downloadCount: `db.downloadToken.update({ where: { token }, data: { downloadCount: { increment: 1 } } })`
  9. Return file stream with headers:
     - `Content-Type`: from blob contentType
     - `Content-Disposition`: `attachment; filename="${product.digitalFileName ?? 'print.png'}"`
     - `X-Content-Type-Options: nosniff`
     - `Cache-Control: private, no-store`
  - If BLOB_PRIVATE_READ_WRITE_TOKEN is not configured, return 503 with message

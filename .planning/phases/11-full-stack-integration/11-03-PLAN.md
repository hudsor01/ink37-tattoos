    Also add mediaQueryOptions and ordersQueryOptions if not already present, following the same pattern as customers/appointments/sessions.

    **Step 2: Convert each dashboard server page**

    For each page (customers, appointments, sessions, media, orders), apply this transformation:

    BEFORE (current pattern):
    ```typescript
    export default async function CustomersPage() {
      const customers = await getCustomers();
      const serialized = JSON.parse(JSON.stringify(customers));
      return <CustomerListClient initialCustomers={serialized} />;
    }
    ```

    AFTER (HydrationBoundary pattern):
    ```typescript
    import { dehydrate, HydrationBoundary, QueryClient } from '@tanstack/react-query';

    export default async function CustomersPage() {
      const queryClient = new QueryClient({ defaultOptions: { queries: { staleTime: 60 * 1000 } } });
      await queryClient.prefetchQuery({
        queryKey: ['customers'],
        queryFn: () => getCustomers(),
      });

      return (
        <HydrationBoundary state={dehydrate(queryClient)}>
          <CustomerListClient />
        </HydrationBoundary>
      );
    }
    ```

    **Step 3: Update each client component**

    Remove `initialData` and `initialXxx` props. The useQuery call now picks up data from the hydrated cache automatically:

    BEFORE:
    ```typescript
    export function CustomerListClient({ initialCustomers }: { initialCustomers: Customer[] }) {
      const { data: customers = [] } = useQuery({
        ...customersQueryOptions,
        initialData: initialCustomers,
      });
    ```

    AFTER:
    ```typescript
    export function CustomerListClient() {
      const { data: customers = [] } = useQuery(customersQueryOptions);
    ```

    Remove the `initialXxx` prop from the component interface entirely. Keep all other component logic unchanged.

    Apply this to all 5 list pages: customers, appointments, sessions, media, orders.

    **IMPORTANT**: The JSON.parse(JSON.stringify()) serialization in server pages is no longer needed because dehydrate() handles serialization automatically.

    **Part A: arrayContains for gallery tag filtering**

    In src/lib/dal/designs.ts, update `getPublicDesigns` (or equivalent gallery data function) to accept optional tag filters and use Drizzle's `arrayContains` operator:

    ```typescript
    import { arrayContains } from 'drizzle-orm';

    export const getPublicDesigns = cache(async (filters?: { tags?: string[] }) => {
      const conditions = [eq(designMedia.isPublic, true)];

      // Only add tag filter if tags are provided and non-empty
      if (filters?.tags && filters.tags.length > 0) {
        conditions.push(arrayContains(designMedia.tags, filters.tags));
      }

      return db.query.designMedia.findMany({
        where: and(...conditions),
        orderBy: [desc(designMedia.createdAt)],
      });
    });
    ```

    IMPORTANT: Guard against empty arrays -- `arrayContains(column, [])` throws in Drizzle. Always check `tags.length > 0` before adding the condition.

    Check the actual schema.ts to find the correct table/column names for the media/design table with tags. The column might be named `tags` on the `designMedia` or `media` table. Read schema.ts first.

    **Part B: .prepare() on hot-path DAL queries**

    Add prepared statements for the most frequently called queries. These go in the DAL files at module scope (not inside the cached function):

    In src/lib/dal/analytics.ts -- prepare the dashboard stat count queries:
    ```typescript
    const customerCountStmt = db.select({ count: count() }).from(customer).prepare('customer_count');
    const appointmentCountStmt = db.select({ count: count() }).from(appointment).prepare('appointment_count');
    const completedSessionCountStmt = db.select({ count: count() })
      .from(tattooSession)
      .where(eq(tattooSession.status, 'COMPLETED'))
      .prepare('completed_session_count');
    const revenueStmt = db.select({ total: sum(tattooSession.totalCost) })
      .from(tattooSession)
      .where(eq(tattooSession.status, 'COMPLETED'))
      .prepare('total_revenue');
    ```

    Then update `getDashboardStats` to use the prepared statements:
    ```typescript
    const [totalCustomersResult, totalAppointmentsResult, ...] = await Promise.all([
      customerCountStmt.execute(),
      appointmentCountStmt.execute(),
      completedSessionCountStmt.execute(),
      revenueStmt.execute(),
      // The findMany with relations can stay as-is (not easily preparable with relations)
    ]);
    ```

    NOTE: The `.prepare()` name argument must be unique across the application. Use descriptive names. Prepared statements work with the WebSocket-based neon driver (pg + @neondatabase/serverless) because the connection can be reused within a request.

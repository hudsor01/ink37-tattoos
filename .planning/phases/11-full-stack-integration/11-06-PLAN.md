    **Part B: Appointment proximity with differenceInDays**

    In the appointment list client, show proximity indicators:

    ```typescript
    import { differenceInDays, differenceInHours } from 'date-fns';

    // For each upcoming appointment, calculate proximity:
    function getProximityLabel(scheduledDate: Date): string | null {
      const now = new Date();
      const daysUntil = differenceInDays(scheduledDate, now);

      if (daysUntil < 0) return null; // past
      if (daysUntil === 0) {
        const hoursUntil = differenceInHours(scheduledDate, now);
        if (hoursUntil <= 0) return null; // past
        return `In ${hoursUntil}h`;
      }
      if (daysUntil === 1) return 'Tomorrow';
      if (daysUntil <= 7) return `In ${daysUntil} days`;
      return null; // too far out to show
    }
    ```

    Render proximity labels as badges next to the scheduled date in the appointment table. Use Badge component with appropriate variant: `destructive` for "Today"/"Tomorrow", `secondary` for "In X days".

    **Part C: isWithinInterval for scheduling conflict detection in DAL**

    In src/lib/dal/appointments.ts, add a helper to check for scheduling conflicts:

    ```typescript
    import { isWithinInterval, addHours } from 'date-fns';

    export async function checkSchedulingConflict(
      proposedDate: Date,
      durationHours: number = 2
    ): Promise<boolean> {
      await requireStaffRole();
      const proposedEnd = addHours(proposedDate, durationHours);

      const existing = await db.query.appointment.findMany({
        where: and(
          eq(appointment.status, 'SCHEDULED'),
          // Check if any existing appointment overlaps
        ),
        columns: { scheduledDate: true },
      });

      return existing.some(appt =>
        isWithinInterval(appt.scheduledDate, { start: proposedDate, end: proposedEnd })
      );
    }
    ```

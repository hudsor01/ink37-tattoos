Create `src/components/dashboard/responsive-data-table.tsx` (per D-04):

`'use client'` directive required.

**MobileField type:**
```typescript
export interface MobileField<TData> {
  label: string;
  accessor: (row: TData) => React.ReactNode;
}
```

**ResponsiveDataTable props** extend DataTable props with:
- `mobileFields: MobileField<TData>[]` -- which 3-4 fields to show in card view
- `mobileActions?: (row: TData) => React.ReactNode` -- optional action buttons/dropdown per card
- All existing DataTable props passed through

**Logic:**
1. Import and use `useIsMobile()` from `@/hooks/use-mobile`
2. If NOT mobile, render `<DataTable {...tableProps} />` passing through all DataTable props
3. If mobile, render a card-based layout:
   - Search bar at top (same as DataTable, using SearchInput if searchKey/globalSearch provided)
   - List of Cards, one per data item
   - Each Card contains:
     - CardContent with the mobileFields rendered as flex rows: `<div className="flex justify-between py-1.5 text-sm"><span className="text-muted-foreground">{field.label}</span><span className="text-right">{field.accessor(row)}</span></div>`
     - If `mobileActions` provided, render at bottom of card with a separator
   - Simple pagination at bottom (matching DataTable pagination style)
   - Handle search/filter client-side for mobile view too (use the same globalSearch or searchKey filtering)

**Key details:**
- Card view implements client-side pagination (same pageSize as DataTable)
- The search input works on mobile by filtering data client-side
- Cards use `Card` from `@/components/ui/card` for consistent styling
- Each card has a subtle bottom border or gap between cards (`space-y-3`)
- The component is generic: `ResponsiveDataTable<TData, TValue>`
- Use `React.useState` for mobile pagination state and search state
- Add `aria-label="Data list"` on the card container for accessibility

**Mobile card pagination:**
- Show "Showing X of Y" text
- Previous/Next buttons matching DataTable style
- Use same pageSize prop

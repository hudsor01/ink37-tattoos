**Part A: Chart accessibility (UI-05, per D-12)**

In `src/components/dashboard/analytics-chart.tsx`, wrap each chart in a `<figure>` element with accessibility attributes:

For each chart component (RevenueChart, AppointmentTypeChart, ClientAcquisitionChart, RevenueComposedChart, BookingTrendsChart):

1. Wrap the `<ChartContainer>` in a `<figure>` element
2. Add `role="img"` to the figure
3. Add a descriptive `aria-label` that summarizes the data. Since the data is dynamic, construct the label from the data prop:
   - RevenueChart: `aria-label={`Revenue chart showing ${data.length} months of data`}`
   - AppointmentTypeChart: `aria-label={`Appointment types breakdown: ${data.map(d => d.type).join(', ')}`}`
   - ClientAcquisitionChart: `aria-label={`Client acquisition over ${data.length} months`}`
   - RevenueComposedChart: `aria-label={`Revenue and session count over ${data.length} months`}`
   - BookingTrendsChart: `aria-label={`Booking trends over ${data.length} weeks`}`
4. Add a visually-hidden summary inside a `<figcaption className="sr-only">` that provides the data in text form. For example:
   ```tsx
   <figcaption className="sr-only">
     {data.map(d => `${d.month}: $${d.revenue.toLocaleString()}`).join(', ')}
   </figcaption>
   ```

**Part B: Interactive element ARIA labels (UI-05)**

Scan each file and add missing ARIA attributes:

1. **data-table.tsx**:
   - Sort buttons: `aria-label={`Sort by ${column name}`}` (the button wrapping the header)
   - View dropdown trigger: `aria-label="Toggle column visibility"`
   - Previous/Next pagination buttons: add `aria-label="Previous page"` and `aria-label="Next page"`
   - Facet filter dropdown triggers: `aria-label={`Filter by ${filter.title}`}`

2. **search-input.tsx**:
   - Input should have `aria-label="Search"` or use the placeholder as label
   - Clear button (if exists): `aria-label="Clear search"`

3. **admin-nav.tsx**:
   - Nav element should have `aria-label="Dashboard navigation"` or `role="navigation"`
   - Each nav link should have descriptive text (likely already has from the link labels)

4. **kpi-card.tsx**:
   - Decorative icons should have `aria-hidden="true"` (they're decorative when next to text)

5. **Forms** (customer, appointment, session, product):
   - All icon-only buttons need `aria-label` or `<span className="sr-only">` text
   - Check for any icon buttons without text labels

6. **Dashboard overview page** (page.tsx):
   - Table in the "Recent Appointments" card: add `aria-label="Recent appointments"` to the Table element

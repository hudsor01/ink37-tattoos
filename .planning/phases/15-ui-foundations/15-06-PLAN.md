  - The payments page is a server component that renders `<DataTable>` directly. DataTable is also a client component. So the pattern is the same -- server components CAN render client components by importing them.
  - HOWEVER, `mobileFields` uses JSX `<StatusBadge>` in accessor functions, which creates a render function. This accessor is a function that returns ReactNode. Server components CAN pass ReactNode (JSX) as props to client components. But the accessor is a function, and functions cannot be passed from server to client components.
  - **Solution:** The payments page must either (a) move the DataTable rendering into a client wrapper component, OR (b) pass mobileFields without function accessors. Option (a) is cleanest.

This works because Next.js serializes the props. The `columns` are imported from `./columns` which is a 'use client' file. So the columns with render functions are defined in a client module and passed in.
  - **The pattern that works:** Define `mobileFields` in the same 'use client' columns file (or a new client file), then import and pass them. OR create a thin client wrapper.

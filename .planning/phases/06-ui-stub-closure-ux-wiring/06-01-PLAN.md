# Plan 06-01: Fix Admin Sign-Out + Payment Success Portal Link

<objective>
Close the two remaining v1.0 audit gaps: (1) admin sign-out uses a GET link instead of Better Auth's signOut() POST action, and (2) the payment success page lacks a link to the client portal where payment history and receipts are visible.
</objective>

## Tasks

### Task 1: Fix admin sign-out to use signOut() POST action

<read_first>
- src/components/dashboard/admin-nav.tsx (current broken sign-out at lines 104-110)
- src/components/portal/portal-header.tsx (reference implementation of correct sign-out pattern)
- src/lib/auth-client.ts (exports signOut function)
</read_first>

<action>
In `src/components/dashboard/admin-nav.tsx` (already a `'use client'` component):

1. Add import: `import { signOut } from '@/lib/auth-client';`

2. Replace the sign-out SidebarMenuButton (lines 104-110):

FROM:
```tsx
<SidebarMenuButton
  tooltip="Sign Out"
  render={<Link href="/api/auth/sign-out" />}
>
  <LogOut />
  <span>Sign Out</span>
</SidebarMenuButton>
```

TO:
```tsx
<SidebarMenuButton
  tooltip="Sign Out"
  onClick={async () => {
    await signOut();
    window.location.href = '/login';
  }}
>
  <LogOut />
  <span>Sign Out</span>
</SidebarMenuButton>
```

3. Remove the `Link` import if it's no longer used elsewhere in the file. Check if any other element uses `Link` — if yes, keep the import.
</action>

<acceptance_criteria>
- `src/components/dashboard/admin-nav.tsx` contains `import { signOut } from '@/lib/auth-client'`
- `src/components/dashboard/admin-nav.tsx` contains `await signOut()` inside an onClick handler
- `src/components/dashboard/admin-nav.tsx` contains `window.location.href = '/login'` after signOut call
- `src/components/dashboard/admin-nav.tsx` does NOT contain `href="/api/auth/sign-out"`
- File still has `'use client'` directive on line 1
</acceptance_criteria>

### Task 2: Add portal link to payment success page

<read_first>
- src/app/(public)/payment/success/page.tsx (current page with only "Return to Home" button)
- src/app/(public)/payment/cancelled/page.tsx (sibling page for style reference)
- src/components/ui/button.tsx (Button component API)
</read_first>

<action>
In `src/app/(public)/payment/success/page.tsx`:

1. Add a secondary link below the existing "Return to Home" button. The page currently has:

```tsx
<Button render={<Link href="/" />}>Return to Home</Button>
```

Add after it:

```tsx
<Button variant="outline" render={<Link href="/portal/payments" />}>
  View Payment History
</Button>
```

2. Wrap both buttons in a flex container with a gap:

```tsx
<div className="flex flex-col gap-3 sm:flex-row">
  <Button render={<Link href="/" />}>Return to Home</Button>
  <Button variant="outline" render={<Link href="/portal/payments" />}>
    View Payment History
  </Button>
</div>
```

This replaces the standalone Button that was there before. The portal link works for all users — unauthenticated users will be redirected to login by the middleware protecting /portal routes.
</action>

<acceptance_criteria>
- `src/app/(public)/payment/success/page.tsx` contains `href="/portal/payments"`
- `src/app/(public)/payment/success/page.tsx` contains text `View Payment History`
- `src/app/(public)/payment/success/page.tsx` contains `variant="outline"` on the portal button
- Both buttons are wrapped in a flex container with `gap-3`
- The "Return to Home" button still exists and links to "/"
</acceptance_criteria>

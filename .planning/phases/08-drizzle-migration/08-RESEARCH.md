## Executive Summary

**Replace Prisma 7 with Drizzle ORM 0.45.1. Keep Neon PostgreSQL. Keep Better Auth (raw pg.Pool adapter).**

23 agents evaluated 15 alternatives across ORM, database provider, and auth dimensions. Drizzle ORM is the clear winner. The codebase has 82 Prisma queries across 20 files, 3 transactions, 19 models, 8 enums, and 16 Decimal columns — all mappable to Drizzle with moderate effort.

## Architecture Patterns

### 1. Single neon-serverless Driver (NOT dual-driver)

Only 3 transactions in the codebase but all are critical (payment/order atomicity). Single driver is simpler — no type mismatch bugs (drizzle-orm#3334), one `db` export, transactions just work.

```typescript
// src/lib/db/index.ts
import 'server-only';
import { drizzle } from 'drizzle-orm/neon-serverless';
import ws from 'ws';
import * as schema from './schema';

export const db = drizzle({
  connection: process.env.DATABASE_URL!,
  ws: ws,
  schema,
});

export type Database = typeof db;
```

### 2. Better Auth via Raw pg.Pool (NOT drizzleAdapter)

Decouples Better Auth from Drizzle version entirely. Uses Kysely internally — richest feature set (programmatic migrations, joins). Auth tables managed by `npx auth migrate`, business tables by `drizzle-kit migrate`.

```typescript
// src/lib/auth.ts
import 'server-only';
import { betterAuth } from 'better-auth';
import { nextCookies } from 'better-auth/next-js';
import { admin } from 'better-auth/plugins';
import { Pool } from 'pg';
import { db } from '@/lib/db';
import * as schema from '@/lib/db/schema';
import { eq } from 'drizzle-orm';

const authPool = new Pool({ connectionString: process.env.DATABASE_URL });

export const auth = betterAuth({
  database: authPool,  // Kysely internally, zero Drizzle coupling
  plugins: [nextCookies(), admin({ defaultRole: 'user' })],
  emailAndPassword: { enabled: true },
  databaseHooks: {
    user: {
      create: {
        after: async (user) => {
          // Business queries use Drizzle (not auth adapter)
          const existing = await db.select()
            .from(schema.customer)
            .where(eq(schema.customer.email, user.email))
            .limit(1);
          if (existing[0] && !existing[0].userId) {
            await db.update(schema.customer)
              .set({ userId: user.id })
              .where(eq(schema.customer.id, existing[0].id));
          } else if (!existing[0]) {
            const [firstName, ...rest] = (user.name || 'Client').split(' ');
            await db.insert(schema.customer).values({
              firstName, lastName: rest.join(' ') || '',
              email: user.email, userId: user.id,
            });
          }
        },
      },
    },
  },
  advanced: { database: { generateId: false } },
});
```

## Don't Hand-Roll

1. **Schema from DB** — Use `drizzle-kit pull` to introspect Neon. Don't manually translate 19 models.
2. **Auth schema** — Run `npx auth migrate` for auth tables. Don't define user/session/account/verification manually in Drizzle.
3. **Relations** — drizzle-kit pull does NOT generate relations in stable. Must write manually using `relations()` helper. There are 18 FK relations to define.
4. **Migration files** — Use `drizzle-kit generate`. Don't write SQL by hand.
5. **Type codegen** — Not needed. Drizzle infers types from schema.ts. No `prisma generate` equivalent.
6. **Prisma type replacements** — `Prisma.InputJsonValue` becomes `Record<string, unknown> | null`. `Prisma.DbNull` becomes `null`. Don't create custom type wrappers.

## Common Pitfalls

### P1: Decimal/numeric returns STRING (CRITICAL — 16 columns affected)
Drizzle `numeric()` returns string by default. Code like `payment.amount * 0.5` becomes string concatenation. TypeScript won't catch it.
**Fix:** Use `numeric({ precision: 10, scale: 2, mode: 'number' })` on all monetary columns.

### P2: drizzle-kit pull generates timestamps as mode:'string'
After introspect, ALL timestamp columns get `mode: 'string'`. Must change to `mode: 'date'` or remove mode (date is default).

### P3: Missing .returning() on mutations
Prisma always returns the record. Drizzle returns `{ rowCount }` without `.returning()`. Compiles fine, breaks at runtime.

### P4: pgEnum must be EXPORTED
Unexported pgEnum declarations silently skipped by drizzle-kit generate. Enum TYPE won't exist in DB.

### P5: Relations NOT auto-generated by pull (stable)
Must write all 18 relations manually. Without them, `db.query.*.findMany({ with })` won't work.

### P6: $onUpdate is application-level only
`.$onUpdate(() => new Date())` runs in ORM, not database. Affects 16 models with @updatedAt.

### P7: No findUnique in Drizzle
Use `db.query.model.findFirst({ where: eq(model.id, id) })`.

### P8: Prisma $transaction([...]) batch pattern
Drizzle has no batch transaction shorthand. Use `db.transaction(async (tx) => { await tx...; await tx...; })`.

### P9: Prisma increment/decrement
`{ paidAmount: { increment: amount } }` becomes `sql\`${table.paidAmount} + ${amount}\``.

### P10: Prisma _count on relations
`include: { _count: { select: { orderItems: true } } }` needs a Drizzle subquery or separate count query.

### P11: Prisma error code P2025
`gift-cards.ts` catches P2025 (record not found). Drizzle doesn't throw on missing records — findFirst returns undefined.

### P12: insensitive mode search
`mode: 'insensitive'` in Prisma where becomes `ilike()` operator in Drizzle.

## Code Examples

### Query Pattern Mapping

| Prisma | Drizzle |
|--------|---------|
| `db.model.findMany({ where, include, orderBy, skip, take })` | `db.query.model.findMany({ where: eq(...), with: {...}, orderBy: [desc(...)], offset, limit })` |
| `db.model.findUnique({ where: { id } })` | `db.query.model.findFirst({ where: eq(model.id, id) })` |
| `db.model.create({ data })` | `db.insert(model).values(data).returning()` |
| `db.model.update({ where, data })` | `db.update(model).set(data).where(eq(...)).returning()` |
| `db.model.delete({ where })` | `db.delete(model).where(eq(...)).returning()` |
| `db.model.upsert({ where, create, update })` | `db.insert(model).values(data).onConflictDoUpdate({ target, set })` |
| `db.model.count({ where })` | `db.select({ count: sql\`cast(count(*) as integer)\` }).from(model).where(...)` |
| `db.model.aggregate({ _sum: { amount } })` | `db.select({ total: sql\`sum(${model.amount})\` }).from(model)` |
| `db.model.groupBy({ by: ['status'], _count })` | `db.select({ status: model.status, count: sql\`cast(count(*) as int)\` }).from(model).groupBy(model.status)` |
| `db.$transaction(async tx => {...})` | `db.transaction(async (tx) => {...})` |
| `db.$transaction([p1, p2])` | `db.transaction(async (tx) => { await tx...; await tx...; })` |
| `{ amount: { increment: val } }` | `set({ amount: sql\`${model.amount} + ${val}\` })` |
| `include: { rel: { select: {...} } }` | `with: { rel: { columns: {...} } }` |
| `where: { OR: [{...}, {...}] }` | `where: or(eq(...), eq(...))` |
| `where: { name: { contains: x, mode: 'insensitive' } }` | `where: ilike(model.name, \`%${x}%\`)` |

import { cache } from 'react';
import { eq, desc } from 'drizzle-orm';
import { db } from '@/lib/db';
import { appointments, customer } from '@/lib/db/schema';

export const getAppointments = cache(async () => {
  await requireStaffRole();

## Migration Workflow

```bash
# 1. Install Drizzle, remove Prisma
npm i drizzle-orm @neondatabase/serverless ws
npm i -D drizzle-kit @types/ws
npm un prisma @prisma/client @prisma/adapter-pg

# 2. Create drizzle.config.ts

# 3. Pull schema from live Neon DB
npx drizzle-kit pull

# 4. Refine generated schema:
#    - Move to src/lib/db/schema.ts
#    - Change timestamps from mode:'string' to mode:'date'
#    - Add mode:'number' to all 16 numeric columns
#    - Write 18 relations() definitions manually
#    - Verify all 8 pgEnums are exported
#    - Replace Prisma types (InputJsonValue, DbNull)

# 5. Validate schema matches DB
npx drizzle-kit generate  # Should produce EMPTY migration

# 6. Create db client (src/lib/db/index.ts)
# 7. Swap auth adapter (src/lib/auth.ts) — raw pg.Pool
# 8. Rewrite 82 queries across 20 files
# 9. Update 2 test files (mock shape change)
# 10. Update 1 test file (schema.test.ts — remove prisma validate)

# 11. Verify
npx next build          # zero errors
npm run test            # all pass
npm audit               # zero high/critical

# 12. Cleanup
rm -rf prisma/ src/generated/prisma/
# Update package.json scripts, CLAUDE.md, PROJECT.md
```

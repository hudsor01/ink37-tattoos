    ```typescript
    import { defineConfig } from "drizzle-kit";
    export default defineConfig({
      dialect: "postgresql",
      schema: "./src/lib/db/schema.ts",
      out: "./drizzle",
      dbCredentials: { url: process.env.DATABASE_URL! },
      introspect: { casing: "camel" },
    });
    ```

    **Step 3: Create `src/lib/db/` directory and run introspect**
    ```bash
    mkdir -p src/lib/db
    npx drizzle-kit pull
    ```
    This generates schema files in `./drizzle/`. Move/adapt the generated table definitions into `src/lib/db/schema.ts`.

    **Step 4: Refine `src/lib/db/schema.ts`** (per D-05, research pitfalls P1-P6)

    The file MUST contain:

    a) **8 exported pgEnum definitions** for: ProductType, OrderStatus, AppointmentStatus, AppointmentType, SessionStatus, ContactStatus, PaymentType, PaymentStatus. Each MUST be exported:
    ```typescript
    export const productTypeEnum = pgEnum('ProductType', ['MERCH', 'PRINT', 'GIFT_CARD']);
    ```

    b) **All 19 table definitions** using `pgTable()`. Table names use snake_case `@@map` names from Prisma schema (e.g., `customers`, `appointments`, `tattoo_sessions`). All IDs are `uuid('id').defaultRandom().primaryKey()`.

    c) **Decimal/numeric columns** — ALL 16 Decimal columns across 6 models MUST use `numeric({ precision: 10, scale: 2, mode: 'number' })` to return numbers not strings (pitfall P1). The affected columns are:
    - TattooSession: totalCost, depositAmount, paidAmount, hourlyRate, estimatedHours, actualHours
    - Order: subtotal, tax, shipping, total
    - OrderItem: price, quantity
    - GiftCard: amount, remainingBalance
    - Payment: amount
    - Product: price
    - TattooArtist: hourlyRate
    - TattooDesign: estimatedCost

    d) **Timestamp columns** — Remove `mode: 'string'` from all introspected timestamp columns (pitfall P2). Use default Date mode. Add `.$onUpdate(() => new Date())` on all `updatedAt` columns (pitfall P6).

    e) **18 FK relations** — Must be written manually (pitfall P5). Use `relations()` helper from `drizzle-orm`. Example:
    ```typescript
    import { relations } from 'drizzle-orm';

    export const customerRelations = relations(customer, ({ many, one }) => ({
      appointments: many(appointment),
      tattooSessions: many(tattooSession),
      payments: many(payment),
      orders: many(order),
      user: one(user, { fields: [customer.userId], references: [user.id] }),
    }));
    ```
    Define relations for ALL models that have foreign keys. Reference `prisma/schema.prisma` for the complete FK map.

    f) **JSON columns** — `auditLog.metadata` and `settings.value` use `jsonb()` type.

    g) **All table and enum exports** — Every `pgTable()` and `pgEnum()` call must be assigned to an exported const. The `schema.ts` file is imported as `* as schema` by the db client.

    **Step 5: Validate schema matches DB**
    ```bash
    npx drizzle-kit generate
    ```
    This should produce an EMPTY migration (no diff between schema.ts and live DB). If it generates SQL, the schema does not match production. Fix discrepancies.

    **Step 1: Create `src/lib/db/index.ts`** (replaces `src/lib/db.ts`)

    ```typescript
    import 'server-only';
    import { drizzle } from 'drizzle-orm/neon-serverless';
    import ws from 'ws';
    import * as schema from './schema';

    export const db = drizzle({
      connection: process.env.DATABASE_URL!,
      ws: ws,
      schema,
    });

    export type Database = typeof db;
    ```

    This uses the single neon-serverless driver per D-01/research. No dual-driver. Transactions work out of the box.

    **Step 2: Delete old `src/lib/db.ts`**

    Remove the file. The new `src/lib/db/index.ts` takes its place. All existing imports of `@/lib/db` will resolve to `src/lib/db/index.ts` automatically (Node resolves directory to index.ts).

    **Step 3: Convert `src/lib/auth.ts`** (per D-09 in research, raw pg.Pool approach)

    Replace the entire file. Key changes:
    - Remove: `import { prismaAdapter } from 'better-auth/adapters/prisma'`
    - Add: `import { Pool } from 'pg'`
    - Add: `import { db } from '@/lib/db'`
    - Add: `import * as schema from '@/lib/db/schema'`
    - Add: `import { eq } from 'drizzle-orm'`
    - Change database config: `database: new Pool({ connectionString: process.env.DATABASE_URL })` (raw Pool, not drizzleAdapter)
    - Convert databaseHooks queries from Prisma to Drizzle:
      - `db.customer.findUnique({ where: { email } })` becomes `db.select().from(schema.customer).where(eq(schema.customer.email, user.email)).limit(1)` then take `[0]`
      - `db.customer.update(...)` becomes `db.update(schema.customer).set({ userId: user.id }).where(eq(schema.customer.id, existing.id))`
      - `db.customer.create(...)` becomes `db.insert(schema.customer).values({ firstName, lastName, email, userId })`
    - Keep ALL other config unchanged: plugins, emailAndPassword, session, user additionalFields, baseURL, secret, advanced.database.generateId
    - Update comment: "Use database UUID generation" instead of "Use Prisma's default UUID generation"

    **Step 4: Re-export schema types for DAL consumers**

    In `src/lib/db/index.ts`, also add:
    ```typescript
    export * from './schema';
    ```
    This allows DAL files to import table references from `@/lib/db` directly if needed, though they can also import from `@/lib/db/schema`.

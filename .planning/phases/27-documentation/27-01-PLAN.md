   Explain: This tells Drizzle the consolidated migration has already been applied. The hash and timestamp come from drizzle/meta/_journal.json. Do NOT run db:migrate on an existing database -- it would try to recreate all tables.

   - Install command: `bun install`
   - Node.js version: 20.x
   - Set all environment variables from Step 1

   - Update DNS records at registrar:
     - A record: `76.76.21.21` (Vercel)
     - CNAME for www: `cname.vercel-dns.com`
   - Wait for DNS propagation (up to 48 hours, usually minutes)
   - Vercel auto-provisions SSL certificate

4. **Project Structure** -- Key directories:
   ```
   src/
     app/
       (public)/     # Public site -- gallery, services, about, contact, FAQ
       (auth)/       # Authentication pages -- login, register
       (dashboard)/  # Admin dashboard -- customers, appointments, analytics
       (portal)/     # Client portal -- appointments, consent, payments
       (store)/      # Online store -- products, cart, checkout
       api/          # Route handlers -- webhooks, health, cron
     lib/
       db/           # Drizzle schema (23 tables), migrations, seed
       auth.ts       # Better Auth config
       dal/          # Data Access Layer -- server-only DB functions
       actions/      # Server Actions -- mutations with auth checks
     components/     # Shared UI components (Shadcn-based)
   docs/             # Operational documentation
   n8n/              # Cron workflow configs
   drizzle/          # Migration files
   ```

5. **Local Development** -- Step-by-step:
   ```bash
   # Clone and install
   git clone https://github.com/hudsor01/ink37-tattoos.git
   cd ink37-tattoos
   bun install

   # Environment
   cp .env.example .env.local
   # Edit .env.local -- see docs/ENV_VARS.md for all variables
   # Minimum for local dev: DATABASE_URL, BETTER_AUTH_SECRET, BETTER_AUTH_URL

   # Database
   bun run db:migrate   # Apply schema
   bun run db:seed      # Seed initial data

   # Run
   bun run dev          # http://localhost:3000
   ```

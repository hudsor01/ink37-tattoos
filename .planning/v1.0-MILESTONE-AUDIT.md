## Tech Debt

4 items across 2 severity levels:

### TD-01: asChild prop on base-ui Button (Moderate)

4 files use `<Button asChild>` which is a Radix UI pattern — base-ui uses `render` prop instead. This produces nested interactive elements (invalid HTML) and potentially broken navigation.

**Files:**
- `src/app/(portal)/portal/no-account/page.tsx` (lines 26, 29)
- `src/app/not-found.tsx` (lines 14, 17)
- `src/app/(dashboard)/dashboard/payments/columns.tsx` (line 72)
- `src/components/dashboard/request-payment-dialog.tsx` (line 78)

**Fix:** Replace `<Button asChild><Link href="...">text</Link></Button>` with `<Button render={<Link href="..." />}>text</Button>`

### TD-02: Orphaned contacts DAL (Low)

`getContacts`, `getContactById`, `updateContactStatus` in `src/lib/dal/contacts.ts` are never imported. No admin contacts page exists. Contact form submissions are stored but invisible to staff.

**Fix:** Create `/dashboard/contacts` page or remove unused exports.

### TD-03: BLOB token env validation (Low)

`src/app/api/store/download/route.ts` reads `process.env.BLOB_PRIVATE_READ_WRITE_TOKEN` directly instead of using the validated `env` object. Missing token causes runtime 503 instead of startup fail-fast.

**Fix:** Add to `env.ts` Zod schema and use `env.BLOB_PRIVATE_READ_WRITE_TOKEN`.

### TD-04: In-memory rate limiter (Info)

`src/lib/security/rate-limiter.ts` uses a module-level `Map` that resets per serverless instance. Provides burst protection within a single process but not across distributed deployment.

**Fix:** Acceptable for v1.0. Consider Redis/Upstash rate limiter for v2 if abuse is observed.

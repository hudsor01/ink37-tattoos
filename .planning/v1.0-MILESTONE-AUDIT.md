## Tech Debt Inventory

### Phase 2: Public Site + Admin Dashboard (2 items)

1. **Home page gallery preview stub** - `src/app/(public)/page.tsx` renders 6 Skeleton elements. Fix: import `getPublicDesigns`, render real design thumbnails.
2. **Dashboard revenue chart stub** - `src/app/(dashboard)/dashboard/page.tsx` shows text placeholder. Fix: import `RevenueChart` from `analytics-chart.tsx`, pass already-fetched `revenueData`.

### Phase 3: Payments (1 item)

1. Tests replicate stripe helper functions instead of importing (server-only workaround) - pragmatic, not a real problem.

### Phase 5: Online Store (4 items)

1. **Order confirmation email download URL wrong** - webhook sends `/store/downloads/${order.id}` but route is `/api/store/download?token=TOKEN`. Customers get 404 from email links.
2. **stripePriceId non-null assertion** - `storeCheckoutAction` crashes if any product lacks Stripe sync. Add guard with user-friendly error.
3. **Gift card purchaser email missing** - Only recipient gets the gift card email. Purchaser receives no confirmation.
4. **Refund handler fragile assumption** - `handleChargeRefunded` calls `db.tattooSession.update` without null-checking `tattooSessionId`. Safe today but fragile for future changes.

### Cross-Phase (4 items)

1. **Admin sign-out GET link** - Should use `signOut()` from auth-client like the portal does.
2. **Payment success page lacks portal link** - Add link to `/portal/payments` after tattoo payment.
3. **getPublicDesignById orphaned** - DAL function exported but never consumed (no gallery detail page).
4. **SUMMARY frontmatter gaps** - 14/20 plan summaries have empty `requirements_completed` field.

**Total: 11 tech debt items across 4 categories**

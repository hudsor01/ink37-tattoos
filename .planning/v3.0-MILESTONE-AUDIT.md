## Unsatisfied Requirements

### INFRA-02: CSP nonce-based headers (Phase 25) — CRITICAL

**What was expected:** proxy.ts generates per-request nonces via `crypto.randomUUID()`, sets `x-nonce` request header, emits dynamic `Content-Security-Policy` response header with nonce in script-src/style-src.

**What exists:** proxy.ts (27 lines) contains only auth redirect logic for /dashboard and /portal routes. No crypto import, no nonce generation, no x-nonce header, no CSP header generation.

**Impact:** `layout.tsx` reads `x-nonce` from headers (always `undefined`). JSON-LD script gets `nonce={undefined}`. No CSP header is sent to browsers. Inline scripts are not nonce-protected.

**Phase 25 VERIFICATION.md (stale):** Claims this was SATISFIED with 8/8 truths verified. This claim is incorrect — the verification was likely generated from plan descriptions rather than actual code inspection.

**Fix required:** Implement nonce generation in middleware (proxy.ts or middleware.ts), set x-nonce header, emit dynamic CSP response header.
